# lib 設計・デバッグ指針

`lib/` はセンサー・画像処理の土台である。この層は実験都合に引っ張られやすいが、本来は上位層が安心して使える安定したインターフェースを提供するべきである。

## 目的

- ハード依存処理の契約を安定させる
- 上位層が値の意味を理解しやすくする
- 視覚系・センサー系の改善を局所化する

## 基本原則

- API の形を安定させる
- エラー表現を一貫させる
- 値そのものと、値の信頼性を分けて扱えるようにする
- デバッグに必要な内部指標を持たせる

## センサー系の指針

- 初期化失敗と読取失敗を区別する
- 値更新失敗時の挙動を明確にする
- 上位で stale 管理しやすい返し方を意識する
- 実機依存の事情をコメントや命名で読み取れるようにする

## 視覚系の指針

- 最終判定だけでなく、判定に至った材料を見えるようにする
- ROI や negative sample の考え方を保守しやすくする
- debug 用の score は「後で閾値調整に使える」ことを意識する

## 避けたい状態

- 実験のたびに戻り値形式が変わる
- 上位でしか意味が分からない隠れた前提がある
- 一時しのぎの例外処理が積み上がる
